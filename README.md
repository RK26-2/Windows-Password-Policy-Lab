**Windows Password Policy Enforcement Lab**

**Overview**

This project demonstrates how to assess password strength and enforce strong password policies using the Windows Local Group Policy Editor. The lab was completed in a safe cloud environment and reflects real SOC Tier 1 defensive practices.


**Objectives**
- Check password strength using Kaspersky’s Password Checker

- Review Windows Local Group Policy Editor

- Configure password policies to enforce strong authentication standards

**Tools Used**

**Key Skills Demonstrated**

- Password policy enforcement

**Lessons Learned**

Completing this lab reinforced several core defensive principles that translate directly into real SOC and incident response work. Even though it was carried out in a safe cloud environment, the workflow mirrored what analysts do when assessing password hygiene, validating policy enforcement, and reducing credential‑related attack surfaces. Weak or predictable passwords remain one of the most common initial access points, and checking password strength highlighted how easily poor credentials can be exploited. Reviewing and configuring password policies through the Local Group Policy Editor showed how endpoint‑level hardening reduces credential‑based attack opportunities and improves the quality of authentication‑related alerts analysts rely on.

The lab also emphasised how important consistent policy enforcement is, as misconfigurations or policy drift can create genuine security gaps during investigations. Finally, documenting each configuration step with clear evidence reflects professional SOC practice, where accurate reporting and reproducible changes are essential for audits, handovers, and post‑incident analysis.


**Conclusion**

This lab strengthened my practical understanding of how enforcing strong password policies directly supports effective SOC and incident response operations.
